02. Backend rdzen i middleware

Analiza plikow odpowiedzialnych za start serwera i kontrole sesji.

Powrot do strony glownej

backend/src/index.js

Plik konfiguruje serwer Express i rejestruje route API.

app.use('/api/auth', authRoutes)
app.use('/api/user', userRoutes)
app.use('/api/reviews', reviewsRoutes)
app.use('/api/spotify', spotifyRoutes)
app.use('/api/playlists', playlistsRoutes)
app.use('/api/recommendations', recommendationsRoutes)

Plik zawiera rowniez endpoint health check oraz middleware globalnej obslugi bledow.

backend/src/middleware/auth.js

Middleware realizuje walidacje tokenu JWT i odswiezanie sesji.

const token = req.cookies.token || req.headers.authorization?.split(' ')[1]
const decoded = jwt.verify(token, process.env.JWT_SECRET)
req.userId = decoded.userId
req.spotifyAccessToken = decoded.spotifyAccessToken

Po wykryciu wygasniecia tokenu wykonywany jest proces refresh token przez Spotify.

backend/src/lib/prisma.js i backend/src/lib/spotify.js

  • prisma.js: singleton PrismaClient.
  • spotify.js: OAuth, wyszukiwanie, top tracks, audio features, playlist sync.
